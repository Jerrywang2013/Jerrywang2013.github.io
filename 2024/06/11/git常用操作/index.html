git常用操作

git clone

克隆项目到本地

git add

将本地变动文件（新加、修改）添加到暂存区

git commit -m “comment”

将暂存区内容提交到本地版本服务器，形成一次提交操作，有对应的日志，后续可以根据此日志来恢复

git push [<远程主机名> [<远程分支名>]]

将本地仓库提交推送到远程。

git push origin 将当前分支推送到远程仓库的同名分支

git push origin <远程分支名> 将当前分支推送到远程仓库的指定分支

如果设置了上游分支，直接用git push就行

git push –set-upstream

git branch –set-upstream-to=/ <本地分支名>

例如：

如果没有设置上游分支，第一次push的时候可以用git push -u origin 来设置，之后就可以直接用git push推送当前分支的修改。

分支

git branch -v | -a

查看分支，本地分支和远程分支颜色显示不同，当前分支颜色也特殊标记

git branch -vv 查看上游分支

git branch -r 查看远程分支

git branch -b <新分支名>

创建分支并且切换到新分支

git checkout -b new-feature origin/feature-branch

从远程分支origin/feature-branch创建本地分支

推送到远程服务器并且创建一个同名的远程分支

git push origin new-feature

创建本地分支后，推送并创建远程分支 -u可以设置上游分支

git push -u origin feature-branch

git checkout new-feature

切换到分支new-feature

git merge new-feature

将new-feature分支内容合并到当前分支。如果有冲突，需要手动编辑冲突文件然后再add | commit | push

git reset head

重置暂存区为，将其恢复到当前分支head指针指向的提交相同的状态。不影响工作区文件，只会影响暂存区且尚未提交的更改。

git rm

从版本控制中删除文件。

例如：将1.txt文件从版本中删除，但是保留在文件系统中

移除版本控制之后，本地工作区文件存在，此时如果切换到其它分支会提示此untracked的工作区文件可能会被重写。主要是其它分支有此文件，切换工作区可能会改写此未被跟踪的文件造成意外情况，解决办法有：

手动备份

move 1.txt 1.txt.bak

git checkout new-feature

暂存未被追踪的文件

强制切换分支

git checkout -f new-feature

此时可能存在未被追踪的文件数据丢失。

git log

显示日志

git relog

基本概念

  • 暂存区（Index/Staging Area）：这是一个临时区域，用于存储即将提交的更改。当你使用 git add 命令时，文件的更改会被添加到暂存区。
  • 工作区（Working Directory）：这是你实际进行开发和修改文件的地方。工作区中的文件可能已经被修改，但还没有被添加到暂存区。
  • HEAD：这是一个指针，指向当前分支的最新提交。
